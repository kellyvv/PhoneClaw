PhoneClaw

PhoneClaw はスマートフォンとエッジデバイス向けのモバイルネイティブなローカル AI Agent フレームワークです。オンデバイスモデル、ネイティブ iOS Skills、権限境界、Live / LiveLand、任意の LAN ベース Mac 推論を組み合わせます。

Core facts

PhoneClaw の役割、機能、データフロー。

ユーザーにも検索エンジンにも読み取りやすい形で、PhoneClaw の主要な事実を整理しています。

01

PhoneClaw とは？

PhoneClaw はスマートフォンとエッジデバイス向けのモバイルネイティブなローカル AI Agent フレームワークです。iOS runtime、オンデバイスモデル、ネイティブ mobile Skills を備えます。

02

何ができますか？

Calendar、Reminders、Contacts、Clipboard、HealthKit、画像理解、音声と LIVE、LiveLand、翻訳、Web Search、Mac Gateway を自然言語で扱えます。

03

データはどう流れますか？

チャット、画像、個人データはデフォルトでデバイス上に残ります。Web Search、URL 読み取り、Mac 推論はユーザーが選ぶ機能です。

04

なぜモバイルネイティブ？

メモリ予算、モデルダウンロード、権限境界、Skill ルーティング、スマートフォン入口、Live interaction、LAN 推論を前提にした runtime です。

Platform labels

Phones, mobile devices, edge devices, and native mobile runtime scenarios across mobile operating systems such as iOS and Android.

Implementation facts

iOS runtime, native iOS Skills, on-device Gemma 4, MiniCPM-V 4.6, Live / LiveLand, and optional LAN Mac Gateway.

Runtime

スマートフォン runtime から設計された Agent stack。

PhoneClaw はスマートフォンを Agent runtime として扱います。モデル読み込み、Skill ルーティング、権限確認、メモリ予算、ダウンロード、Live 入口、タスク状態はモバイル制約を前提に設計されています。

オンデバイスモデル

Gemma 4 E2B / E4B を LiteRT-LM 経由で iPhone 上で実行し、MiniCPM-V 4.6 を画像理解と LIVE カメラ向けに使います。

ネイティブ iOS Skills

Calendar、Reminders、Contacts、Clipboard、HealthKit、翻訳、画像、明示的な Web Search を権限境界内で扱います。

Mac Gateway

より重い推論が必要なときは、同じ LAN 上の Mac を任意の推論ノードとして使えます。iPhone は Agent 体験と権限境界を維持します。

Privacy

データはデバイス側に残ります。

チャット、画像、個人データはデフォルトでデバイス上に残ります。Web Search、URL の読み取り、ペアリングした Mac へのリモート推論は、ユーザーが明示的に使うネットワーク機能です。
