01

PhoneClaw 是什么？

PhoneClaw 是面向手机和端侧设备的移动端本地 AI Agent 框架，提供 iOS runtime、端侧模型和原生移动端 Skills。

02

能做什么？

用自然语言处理日历、提醒事项、通讯录、剪贴板、健康数据摘要、图片理解、语音与 LIVE、LiveLand、翻译、联网搜索和 Mac Gateway。

03

数据怎样处理？

聊天、图片和个人数据默认留在设备端。联网搜索、打开网页和 Mac 远程推理是用户显式触发的能力。

04

为什么是移动端框架？

框架围绕移动端内存预算、模型下载、权限边界、Skill 路由、手机入口、Live 交互和局域网边缘推理设计。

平台分类

手机、移动端设备、端侧设备，以及 iOS / Android 等移动操作系统下的原生运行时场景。

实现事实

iOS runtime、原生 iOS Skills、端侧 Gemma 4、MiniCPM-V 4.6、Live / LiveLand 和可选局域网 Mac Gateway。

运行时

面向手机运行时设计的 Agent stack。

PhoneClaw 把手机当成 Agent 运行时。模型加载、Skill 路由、权限确认、内存预算、模型下载、Live 入口和任务状态都围绕移动端约束设计。

端侧模型

Gemma 4 E2B / E4B 通过 LiteRT-LM 在手机端运行，MiniCPM-V 4.6 用于图片和 LIVE 相机场景。

原生 iOS Skills

Calendar、Reminders、Contacts、Clipboard、HealthKit、翻译、图片和显式 Web Search 都在权限边界内执行。

Mac Gateway

需要更重推理时，可以把同一局域网内的 Mac 作为可选推理节点，iPhone 仍保留 Agent 体验和权限边界。

隐私

数据留在设备端，联网能力显式触发。

聊天、图片和个人数据默认留在设备端。联网搜索、打开网页和配对 Mac 远程推理是用户显式触发的能力。PhoneClaw 的本地路径支持完全离线运行。
