Privacy and data flow

Know exactly where the data goes.

PhoneClaw is local-first. In the local runtime, chat, images, and personal data stay on device by default. Web Search, URL reading, and Mac remote inference are explicit user actions.

View data table Back to home

Short version

The default experience runs on device.

Inference and native Skill calls run on-device by default. Explicit network paths cover public Web Search, URL reading, and paired Mac remote inference.

Default path

Local model, local Skills, local summaries, local permission checks.

Public web path

Web Search and URL reading run when the user selects public web features.

Mac path

Mac remote inference runs after pairing your Mac and selecting a Mac-side model.

Data table

What stays on device.

Permission-scoped iOS data is read after you grant access. Each Skill follows the permission scope provided by iOS.

DataProcessing pathDefault behavior
Chat messages On-device model context and local conversation state. Stays on device
Images and camera input On-device multimodal model and LIVE mode pipeline. Stays on device
Calendar, Reminders, Contacts Native iOS APIs after explicit permission. Stays on device
Clipboard Local task step, useful for multi-step workflows. Stays on device
HealthKit summaries Steps, distance, calories, heart rate, sleep, workouts, and related summaries after permission. Stays on device
Web Search query Sent to a search provider when you explicitly ask for realtime public information. Explicit action
Specific public URL Fetched from the target website when you ask PhoneClaw to read that page. Explicit action
Mac remote inference request Sent to your paired Mac on the same LAN. With Ollama it stays on that Mac; other providers follow their own policy. Opt-in action
