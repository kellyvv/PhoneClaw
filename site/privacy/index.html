PhoneClaw Privacy Policy

PhoneClaw is an account-free local AI agent runtime for phones. This policy explains local data handling, explicit network features, and limited diagnostic data used to improve stability and reliability.

Policy summary Data table

Last updated: July 8, 2026

Summary

PhoneClaw respects your privacy.

The app is built around a local-first runtime. Local AI features run on the device where possible, and network features are user-selected actions.

01

Account-free use

PhoneClaw does not require users to create an account or sign in to use the app.

02

Local AI path

Prompts, AI responses, images, and personal mobile data are handled by the local runtime by default.

03

Limited diagnostics

PhoneClaw may collect non-personal usage and diagnostic data such as app version, build number, device model, OS version, language/region, session events, model download status, and crash or error information.

04

Purpose-limited use

Diagnostic data is used to improve app stability, model download reliability, and the product experience.

Policy

How PhoneClaw handles app data.

This policy applies to the PhoneClaw app, its local AI features, native mobile Skills, explicit Web Search and URL reading, optional Mac Gateway inference, and app diagnostics.

Accounts

PhoneClaw does not require account registration. Diagnostic events are not linked to a PhoneClaw account.

User prompts and AI responses

User prompts and AI responses are processed locally by default and are not used for advertising tracking. When you choose Web Search, URL reading, or Mac Gateway inference, the content needed for that selected feature follows that network path.

Personal mobile data

Calendar, Reminders, Contacts, Clipboard, Health data, photos, camera input, and microphone input are accessed only through the relevant mobile OS permission and requested feature. PhoneClaw diagnostics do not collect these personal content sources.

Analytics and diagnostics

PhoneClaw may collect limited, non-personal analytics and diagnostic information: app version, build number, device model, operating system version, language/region settings, session events, feature events, model download status, performance data, crash reports, error information, and a device or install identifier used for diagnostics. This data is not linked to your identity and is not used for tracking.

Advertising and sale of data

PhoneClaw does not sell user data. PhoneClaw does not use collected data for advertising tracking.

Retention

Diagnostic data is kept for the period reasonably needed to troubleshoot crashes, improve reliability, and understand release quality.

Choices

User choices and controls.

PhoneClaw uses mobile OS permissions for personal data sources. You can manage those permissions in system settings.

Permissions

Mobile OS controls

Calendar, Reminders, Contacts, Health, camera, microphone, and photo access are controlled by the permissions available on your device.

Local mode

On-device path

Local model features and native Skills can run through the phone runtime without using Web Search or Mac Gateway.

Network features

Explicit actions

Web Search, URL reading, and Mac Gateway inference are selected by the user for tasks that need those paths.

Contact

Questions

For privacy questions, contact the developer through GitHub Issues.

中文摘要

PhoneClaw 隐私政策摘要。

PhoneClaw 不要求注册账号。默认本地路径在设备端处理提示词、AI 回复、图片和个人移动端数据。联网搜索、打开网页和 Mac Gateway 是用户显式选择的能力。PhoneClaw 可能收集有限的非个人使用与诊断信息，例如 App 版本、构建版本、设备型号、系统版本、语言区域、会话事件、模型下载状态、崩溃和错误信息，用于改进稳定性、下载可靠性和产品体验。PhoneClaw 不出售用户数据，也不将收集的数据用于广告追踪。
